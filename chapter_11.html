1. Rule 1: The Last Declaration Will Win

When multiple CSS rules apply to the same element, the last declaration in the stylesheet will take precedence and be applied to the element. This means that if there are conflicting styles, the one that appears later in the CSS code will override the earlier ones. This rule is important to understand when writing CSS, as it allows developers to control which styles are applied to elements by carefully ordering their declarations.

Code: 

#te {
  color: blue !important;
}

#te {
  color: red !important;
}

<p id="te">
  This paragraph which fistly i applied in blue color but last time i
  applied in red color. So now this paragraph will be red color
</p>

2. Rule 2: The More Specific Declaration Will Win

When multiple CSS rules apply to the same element, the more specific rule will take precedence over less specific rules. Specificity is a measure of how specific a CSS selector is. For example, an ID selector has higher specificity than a class selector, and a class selector has higher specificity than an element selector. This means that if there are conflicting styles, the one with higher specificity will be applied to the element.

This paragraph will be red because the class selector has higher specificity than the tag selector.

This paragraph will be green because the ID selector has higher specificity than the class selector.

Code: 

.highlight {
  color: red;
}

<div id="secondRule">
  <p class="highlight">
    This paragraph will be red because the class selector has higher
    specificity than the tag selector.
  </p>

  <p id="important" class="highlight">
    This paragraph will be green because the ID selector has higher
    specificity than the class selector.
  </p>
</div>

3. Rule 3: Inline Style Is Preferred to both Internal and External Styles

When multiple CSS rules apply to the same element, inline styles will take precedence over both internal and external styles. Inline styles are defined directly on an HTML element using the style attribute, while internal styles are defined within a style tag in the head of the document, and external styles are defined in a separate CSS file. This means that if there are conflicting styles, the inline style will be applied to the element, regardless of the order of the internal or external styles.

This paragraph will be red due to the inline style, even though there are internal styles set.

Code: 

<span style="color: red">

4. Rule 4: “!important” Declaration Is the Strongest Rule

When multiple CSS rules apply to the same element, the declaration with the "!important" flag will take precedence over all other declarations, regardless of specificity or order. The "!important" flag is used to indicate that a particular style should be applied with the highest priority, and it can be added to any CSS declaration. This means that if there are conflicting styles, the one with the "!important" flag will be applied to the element, even if it is less specific or appears earlier in the stylesheet.
This paragraph will be green due to the !important declaration.

Code: 

<div id="fourthRule">
  <article class="special" id="unique">
    This paragraph will be green due to the !important declaration.
  </article>
</div>

5. Inheritance

In CSS, inheritance refers to the way certain properties are passed down from parent elements to their child elements in the HTML document tree. When a property is inherited, the child element automatically takes on the same value as its parent element, unless it has its own specific value defined. This allows for a more efficient and consistent styling of web pages, as common styles can be defined on parent elements and automatically applied to their children.

Inheritance

This is the parent element. [This blue color and font-family will be inherited by child elements.]
This is the child element. [This font-size 1.2em will be inherited by the grandchild]
This is the grandchild element.

Code: 

/* ID selector */
#important {
  color: green;
}

/* Internal stylesheet */
.special {
  color: green;
}

/* More specific internal stylesheet */
#unique {
  color: purple;
}

/* !important declaration */
article {
  color: green !important;
}

.parent {
  color: blue;
  font-family: Arial, sans-serif;
}

.child {
  font-size: 1.2em;
  font-weight: bold;
}

.grandchild {
  font-size: inherit;
  background-color: yellow;
}

<div class="parent">
  This is the parent element. [This blue color and font-family will be
  inherited by child elements.]
  <div class="child">
    This is the child element. [This font-size 1.2em will be inherited
    by the grandchild]
    <div class="grandchild">
      This is the grandchild element.
    </div>
  </div>
</div>

  <article class="special" id="unique">
    This paragraph will be green due to the !important declaration.
  </article>
</div>
