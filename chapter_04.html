1. Semantic Elements

Semantic elements clearly describe their meaning in a human- and machine-readable way. Examples of semantic elements include <header>, <footer>, <article>, <section>, and <nav>. These elements provide information about the structure and content of a webpage, making it easier for both developers and browsers to understand the layout and purpose of different sections.

Code:

<article>, <aside>, <details>, <figcaption>, 
<figure>, <footer>, <header>, <main>, <mark>, <nav>, <section>, <summary>, <time>

2. Semantic Layout Elements

Semantic layout elements are used to structure the content of a webpage in a meaningful way. For example, <header> is typically used for introductory content or navigation links, while <footer> is used for information about the author, copyright, or related documents. <article> represents a self-contained piece of content that could be distributed independently, and <section> is used to group related content together. Using these elements helps improve the accessibility and SEO of a webpage.

Code:

<header>, <nav>, <main>, <section>, 
<article>, <aside>, <footer>

3. Semantic Formatting Elements

Semantic formatting elements are used to indicate the meaning of the content they contain. For example, <mark> is used to highlight text, <time> is used to represent a specific time or date, and <details> can be used to create an interactive widget that users can open and close to reveal additional information. These elements help convey the purpose of the content and enhance the user experience.

Code:

<strong>, <em>, <mark>, <cite>, 
<blockquote>, <abbr>, <code>, <pre>, <time>
